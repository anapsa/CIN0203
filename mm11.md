## 8.6 — NP-hard/complete
Se o problema geral é NP-hard, não procurar solução polinomial. Procurar n pequeno, pesos pequenos, grafo especial, árvore/DAG/bipartido ou alguma restrição escondida.

**Estratégias comuns**
- backtracking;
- bitmask DP;
- meet in the middle;
- branch and bound;
- DP pseudo-polinomial.

**Problemas para reconhecer**
| Problema | Caminho comum |
|---|---|
| Knapsack | DP item/capacidade |
| Subset Sum | DP por soma ou meet in the middle |
| Coin Change | DP por valor |
| TSP | DP com bitmask |
| Hamiltonian Path | backtracking/bitmask |
| Longest Path | fácil em DAG, difícil no geral |
| Max Independent Set | backtracking/bitmask |
| Min Vertex Cover | em bipartido: matching |
| Set Cover | bitmask/branch and bound |
| SAT | backtracking ou caso especial |
| Steiner Tree | DP se terminais são poucos |
| Graph Coloring | backtracking com poda |

## 8.7 — Problem Decomposition
Problemas difíceis muitas vezes misturam técnicas. Tentar quebrar em partes.

**Padrões comuns**
- Binary Search the Answer + checker: quando a resposta é monotônica ( false, false, false, true, true, true).
- Data Structure + algoritmo: sort + fenwick, DFS + segment tree, sweep line + set.
- Graph preprocessing + DP: primeiro BFS/Dijkstra/Floyd, depois DP.
- Geometry + outra técnica: geometria com busca binária, grafo, ordenação ou estrutura de dados.
- Três ou mais componentes: resolva por camadas, não tentar enxergar tudo de uma vez.
